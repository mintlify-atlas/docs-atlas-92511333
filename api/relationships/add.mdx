## Overview

This endpoint creates a many-to-many relationship between a part and a vehicle. It links the specified vehicle to the specified part, indicating that the part is compatible with or used in that vehicle.

## Endpoint

```
POST /api/v1/parts/{partId}/vehicles/{vehicleId}
```

## Path Parameters

<ParamField path="partId" type="Long" required>
  The unique identifier of the part to which you want to add a vehicle relationship
</ParamField>

<ParamField path="vehicleId" type="Long" required>
  The unique identifier of the vehicle to associate with the part
</ParamField>

## Request

No request body is required. The relationship is created using only the path parameters.

## Response

Returns the updated `Part` object with its full set of associated vehicles, including the newly added one.

## Controller Implementation

The endpoint is implemented in the `PartController` class:

```java
@PostMapping("/parts/{partId}/vehicles/{vehicleId}")
@ApiOperation(value = "Relaciona um veículo a uma peça")
@ResponseStatus(HttpStatus.OK)
public ResponseEntity<Part> addVehicleToPart(
        @PathVariable("partId") Long partId,
        @PathVariable("vehicleId") Long vehicleId) {
    Optional<Part> partData = partRepository.findById(partId);
    Optional<Vehicle> vehicleData = vehicleRepository.findById(vehicleId);

    if (!partData.isPresent() || !vehicleData.isPresent()) {
        return new ResponseEntity<>(HttpStatus.NOT_FOUND);
    }

    Part part = partData.get();
    part.addVehicle(vehicleData.get());
    return new ResponseEntity<>(partRepository.save(part), HttpStatus.OK);
}
```

## Bidirectional Relationship

The `addVehicle` method in the `Part` model maintains bidirectional consistency:

```java
public void addVehicle(Vehicle vehicle) {
    this.vehicles.add(vehicle);
    vehicle.getParts().add(this);
}
```

Reference: `~/workspace/source/backend/src/main/java/br/com/tarcnux/spring/jpa/postgresql/model/Part.java:94-97`

This ensures that:
- The vehicle is added to the part's vehicle set
- The part is added to the vehicle's part set
- Both sides of the relationship are synchronized

## Use Cases

- **Part Compatibility** - Mark a part as compatible with a new vehicle model
- **Catalog Expansion** - Add vehicle relationships as new compatibility information becomes available
- **Inventory Management** - Link parts to vehicles for better stock planning
- **Customer Service** - Build cross-reference data to help customers find parts

## Important Notes

<Note>
  **Duplicate Prevention**: If the relationship already exists, the operation is idempotent and won't create duplicate entries due to the use of `Set<Vehicle>` in the Part model.
</Note>

<Warning>
  **Prerequisite**: Both the part and vehicle must exist in the database before creating a relationship. If either ID is invalid, the endpoint returns `404 NOT_FOUND`.
</Warning>
