## Overview

This endpoint returns a set of all vehicles that are linked to a specific part. It's useful for understanding part compatibility and finding which vehicles can use a particular component.

## Endpoint

```
GET /api/v1/parts/{partId}/vehicles
```

## Path Parameters

<ParamField path="partId" type="Long" required>
  The unique identifier of the part whose vehicle relationships you want to retrieve
</ParamField>

## Response

Returns a `Set<Vehicle>` containing all vehicles associated with the specified part.

### Response Structure

```json
[
  {
    "id": 1,
    "name": "Fusca",
    "manufacturer": "Volkswagen",
    "year": 1970
  },
  {
    "id": 2,
    "name": "Gol",
    "manufacturer": "Volkswagen",
    "year": 2020
  }
]
```

### Success Response

<ResponseField name="200 OK" type="Set<Vehicle>">
  Successfully retrieved the set of vehicles. Returns an empty set if no vehicles are associated with the part.
</ResponseField>

### Error Responses

<ResponseField name="404 NOT_FOUND" type="Error">
  The specified part ID does not exist in the database.
</ResponseField>

## Controller Implementation

The endpoint is implemented in the `PartController` class:

```java
@GetMapping("/parts/{partId}/vehicles")
@ApiOperation(value = "Lista os veículos relacionados a uma peça")
@ResponseStatus(HttpStatus.OK)
public ResponseEntity<Set<Vehicle>> getVehiclesByPart(@PathVariable("partId") Long partId) {
    Optional<Part> partData = partRepository.findById(partId);
    if (!partData.isPresent()) {
        return new ResponseEntity<>(HttpStatus.NOT_FOUND);
    }
    return new ResponseEntity<>(partData.get().getVehicles(), HttpStatus.OK);
}
```

- **Inventory Planning** - Understand the demand scope for a particular part
- **Cross-Reference** - Build compatibility tables for parts catalogs
