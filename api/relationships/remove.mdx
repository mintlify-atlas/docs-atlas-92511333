## Overview

This endpoint removes the many-to-many relationship between a part and a vehicle. It unlinks the specified vehicle from the specified part, indicating that the part is no longer associated with that vehicle.

## Endpoint

```
DELETE /api/v1/parts/{partId}/vehicles/{vehicleId}
```

## Path Parameters

<ParamField path="partId" type="Long" required>
  The unique identifier of the part from which you want to remove a vehicle relationship
</ParamField>

<ParamField path="vehicleId" type="Long" required>
  The unique identifier of the vehicle to disassociate from the part
</ParamField>

## Request

No request body is required. The relationship is removed using only the path parameters.

## Response

Returns the updated `Part` object with its remaining set of associated vehicles, excluding the removed one.

### Success Response

<ResponseField name="200 OK" type="Part">
  Successfully removed the relationship. Returns the updated part with remaining associated vehicles.
</ResponseField>

### Error Responses

<ResponseField name="404 NOT_FOUND" type="Error">
  Either the specified part ID or vehicle ID does not exist in the database.
</ResponseField>

## Controller Implementation

The endpoint is implemented in the `PartController` class:

```java
@DeleteMapping("/parts/{partId}/vehicles/{vehicleId}")
@ApiOperation(value = "Remove o relacionamento entre veículo e peça")
@ResponseStatus(HttpStatus.OK)
public ResponseEntity<Part> removeVehicleFromPart(
        @PathVariable("partId") Long partId,
        @PathVariable("vehicleId") Long vehicleId) {
    Optional<Part> partData = partRepository.findById(partId);
    Optional<Vehicle> vehicleData = vehicleRepository.findById(vehicleId);

    if (!partData.isPresent() || !vehicleData.isPresent()) {
        return new ResponseEntity<>(HttpStatus.NOT_FOUND);
    }

    Part part = partData.get();
    part.removeVehicle(vehicleData.get());
    return new ResponseEntity<>(partRepository.save(part), HttpStatus.OK);
}
```

## Bidirectional Relationship Cleanup

The `removeVehicle` method in the `Part` model maintains bidirectional consistency:

```java
public void removeVehicle(Vehicle vehicle) {
    this.vehicles.remove(vehicle);
    vehicle.getParts().remove(this);
}
```

Reference: `~/workspace/source/backend/src/main/java/br/com/tarcnux/spring/jpa/postgresql/model/Part.java:99-102`

This ensures that:
- The vehicle is removed from the part's vehicle set
- The part is removed from the vehicle's part set
- Both sides of the relationship are properly cleaned up
- No orphaned relationships remain in the database

## Use Cases

- **Compatibility Update** - Remove outdated or incorrect vehicle associations
- **Part Discontinuation** - Unlink parts that are no longer compatible with certain vehicles
- **Data Correction** - Fix mistaken relationships in the catalog
- **Inventory Management** - Update part-vehicle associations as products change

## Important Notes

<Note>
  **Idempotent Operation**: If the relationship doesn't exist, the operation will complete without error. The part is saved regardless, ensuring database consistency.
</Note>

<Warning>
  **Prerequisite**: Both the part and vehicle must exist in the database. If either ID is invalid, the endpoint returns `404 NOT_FOUND`.
</Warning>

<Info>
  **Soft Relationship**: Removing a relationship does NOT delete the part or vehicle entities. It only removes the association between them. Both entities remain in their respective tables.
</Info>
