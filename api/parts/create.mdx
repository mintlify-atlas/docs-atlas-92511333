## Endpoint

```
POST http://localhost:8080/api/v1/parts
```

Creates a new part in the catalog and returns the created part with an auto-generated ID.

## Request Body

<ParamField body="name" type="string" required>
  Name of the part. This field is required.
</ParamField>

<ParamField body="applicationVehicle" type="string">
  Description of the vehicle application for this part (e.g., "Fusca", "Gol").
</ParamField>

<ParamField body="netWeight" type="double">
  Net weight of the part. Must be a decimal number.
</ParamField>

<ParamField body="grossWeight" type="double">
  Gross weight of the part. Must be a decimal number.
</ParamField>

## Response

<ResponseField name="id" type="long">
  Auto-generated unique identifier for the created part
</ResponseField>

<ResponseField name="name" type="string">
  Name of the part
</ResponseField>

<ResponseField name="applicationVehicle" type="string">
  Vehicle application description
</ResponseField>

<ResponseField name="netWeight" type="double">
  Net weight of the part
</ResponseField>

<ResponseField name="grossWeight" type="double">
  Gross weight of the part
</ResponseField>

## Status Codes

| Status Code | Description |
|-------------|-------------|
| 201 Created | Part was successfully created |
| 500 Internal Server Error | An error occurred while creating the part |

## Controller Implementation

The endpoint is implemented in the `PartController` class:

```java PartController.java:86-98
@PostMapping("/parts")
@ApiOperation(value = "Cadastra uma peça")
@ResponseStatus(HttpStatus.CREATED)
@ResponseBody
public ResponseEntity<Part> createPart(@RequestBody Part part) {
    try {
        Part _part = partRepository.save(new Part(part.getName(), 
            part.getApplicationVehicle(), 
            part.getNetWeight(),
            part.getGrossWeight()));
        return new ResponseEntity<Part>(_part, HttpStatus.CREATED);
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

## Notes

- The `id` field is auto-generated by the database and should not be included in the request body
- All fields except `name` are optional but recommended for complete part information
- Weight values should be decimal numbers (e.g., 9.87)
- The endpoint uses Spring Data JPA to persist the part to a PostgreSQL database
