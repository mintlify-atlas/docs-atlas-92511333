## Overview

The search functionality is built into the main GET endpoint and allows you to find parts by keyword matching on either the part name or the vehicle application field.

## Search by Part Name

```
GET http://localhost:8080/api/v1/parts?name={keyword}
```

Searches for parts where the name contains the specified keyword (case-insensitive). Results are returned in alphabetical order by name.

### Query Parameters

<ParamField query="name" type="string" required>
  Keyword to search for in part names. The search is case-insensitive and matches partial strings.
</ParamField>

### Response

Returns an array of Part objects matching the search criteria.

<ResponseField name="id" type="long">
  Unique identifier for the part
</ResponseField>

<ResponseField name="name" type="string">
  Name of the part (contains the search keyword)
</ResponseField>

<ResponseField name="applicationVehicle" type="string">
  Vehicle application description
</ResponseField>

<ResponseField name="netWeight" type="double">
  Net weight of the part
</ResponseField>

<ResponseField name="grossWeight" type="double">
  Gross weight of the part
</ResponseField>

### Status Codes

| Status Code | Description |
|-------------|-------------|
| 200 OK | Parts found matching the search criteria |
| 204 No Content | No parts found matching the search criteria |
| 500 Internal Server Error | An error occurred |

### Example Request

curl -X GET "http://localhost:8080/api/v1/parts?name=polia"

## Controller Implementation

The search logic is implemented in the `getAllParts` method:

```java PartController.java:43-71
@GetMapping("/parts")
@ApiOperation(value = "Exibe todas as peças em ordem alfabética")
@ResponseStatus(HttpStatus.OK)
public ResponseEntity<List<Part>> getAllParts(
        @RequestParam(required = false) String name,
        @RequestParam(required = false) String vehicle) {
    
    try {
        List<Part> parts = new ArrayList<Part>();
        
        if (name == null && vehicle == null) {
            partRepository.findByOrderByNameAsc().forEach(parts::add);
        } else if (vehicle == null) {
            partRepository.findByNameContainingIgnoreCaseOrderByNameAsc(name)
                .forEach(parts::add);
        } else {
            partRepository.findByApplicationVehicleContainingIgnoreCaseOrderByNameAsc(vehicle)
                .forEach(parts::add);
        }
        
        if (parts.isEmpty()) {
            return new ResponseEntity<>(HttpStatus.NO_CONTENT);
        }
        
        return new ResponseEntity<List<Part>>(parts, HttpStatus.OK);
        
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

## Search Behavior

<Info>
  **Search Priority**: When both `name` and `vehicle` parameters are provided, the `vehicle` parameter takes precedence and the `name` parameter is ignored.
</Info>

### Key Features

1. **Case-Insensitive**: Searches are case-insensitive, so "fusca", "Fusca", and "FUSCA" will all return the same results
2. **Partial Matching**: The search matches any part of the field (substring matching)
3. **Alphabetical Results**: All search results are sorted alphabetically by part name
4. **No Results**: Returns 204 No Content if no parts match the search criteria

## Tips for Effective Searching

1. **Use partial keywords**: You don't need to type the complete word. "alt" will match "Alternador"
2. **Case doesn't matter**: Search is case-insensitive, so search however is comfortable for you
3. **Empty results**: A 204 status code means no matches were found, not an error
4. **Special characters**: URL-encode special characters in query parameters
