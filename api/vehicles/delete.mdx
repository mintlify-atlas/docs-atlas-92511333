## Delete Single Vehicle

```
DELETE /api/v1/vehicles/{id}
```

Deletes a specific vehicle by its unique identifier.

### Path Parameters

<ParamField path="id" type="long" required>
  The unique identifier of the vehicle to delete
</ParamField>

### Example Request

```bash
curl -X DELETE http://localhost:8080/api/v1/vehicles/1
```

### Response

Returns no content on successful deletion.

### Status Codes

<ResponseField name="204" type="No Content">
  Vehicle successfully deleted
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  An error occurred while deleting the vehicle (e.g., vehicle does not exist or database error)
</ResponseField>

### Controller Implementation

```java
@DeleteMapping("/vehicles/{id}")
@ApiOperation(value = "Exclui um veículo")
@ResponseStatus(HttpStatus.NO_CONTENT)
public ResponseEntity<HttpStatus> deleteVehicle(@PathVariable("id") Long id) {
    try {
        vehicleRepository.deleteById(id);
        return new ResponseEntity<>(HttpStatus.NO_CONTENT);
    } catch (Exception e) {
        return new ResponseEntity<>(HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

## Delete All Vehicles

```
DELETE /api/v1/vehicles
```

Deletes all vehicles from the catalog.

<Warning>
  This operation cannot be undone. Use with caution.
</Warning>

### Example Request

```bash
curl -X DELETE http://localhost:8080/api/v1/vehicles
```

### Response

Returns no content on successful deletion.

### Status Codes

<ResponseField name="204" type="No Content">
  All vehicles successfully deleted
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  An error occurred while deleting vehicles
</ResponseField>

### Controller Implementation

```java
@DeleteMapping("/vehicles")
@ApiOperation(value = "Exclui todos os veículos")
@ResponseStatus(HttpStatus.NO_CONTENT)
public ResponseEntity<HttpStatus> deleteAllVehicles() {
    try {
        vehicleRepository.deleteAll();
        return new ResponseEntity<>(HttpStatus.NO_CONTENT);
    } catch (Exception e) {
        return new ResponseEntity<>(HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

**Source:** `VehicleController.java:108-118`

## Notes

- Deleting a vehicle does not automatically delete associated parts
- The many-to-many relationship entries between vehicles and parts are handled by the database
- If a vehicle is associated with parts, the relationship entries will be removed
- Both operations return `204 No Content` on success with no response body
