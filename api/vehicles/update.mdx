## Endpoint

```
PUT /api/v1/vehicles/{id}
```

## Path Parameters

<ParamField path="id" type="long" required>
  The unique identifier of the vehicle to update
</ParamField>

## Request Body

<ParamField body="name" type="string" required>
  The new name for the vehicle
</ParamField>

### Example Request Body

```json
{
  "name": "Fusca 1970"
}
```

## Response

Returns the updated vehicle object.

<ResponseField name="id" type="long">
  The unique identifier of the vehicle
</ResponseField>

<ResponseField name="name" type="string">
  The updated name of the vehicle
</ResponseField>

### Example Response

**Status Code:** `200 OK`

```json
{
  "id": 1,
  "name": "Fusca 1970"
}
```

## Status Codes

<ResponseField name="200" type="OK">
  Vehicle successfully updated
</ResponseField>

<ResponseField name="404" type="Not Found">
  Vehicle with the specified ID does not exist
</ResponseField>

## cURL Example

```bash
curl -X PUT http://localhost:8080/api/v1/vehicles/1 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Fusca 1970"
  }'
```

## Controller Implementation

The update vehicle endpoint is implemented in the `VehicleController` class:

```java
@PutMapping("/vehicles/{id}")
@ApiOperation(value = "Atualiza um veículo")
@ResponseStatus(HttpStatus.OK)
@ResponseBody
public ResponseEntity<Vehicle> updateVehicle(@PathVariable("id") Long id, @RequestBody Vehicle vehicle) {
    Optional<Vehicle> vehicleData = vehicleRepository.findById(id);
    if (vehicleData.isPresent()) {
        Vehicle _vehicle = vehicleData.get();
        _vehicle.setName(vehicle.getName());
        return new ResponseEntity<>(vehicleRepository.save(_vehicle), HttpStatus.OK);
    }
    return new ResponseEntity<>(HttpStatus.NOT_FOUND);
}
```

**Source:** `VehicleController.java:82-94`

## Notes

- The vehicle must exist in the database to be updated
- Only the name field can be updated through this endpoint
- The ID cannot be changed
- Relationships with parts are preserved when updating a vehicle
