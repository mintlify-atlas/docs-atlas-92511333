## Endpoint

```
POST /api/v1/vehicles
```

## Request Body

<ParamField body="name" type="string" required>
  The name of the vehicle to create (e.g., "Fusca", "Gol", "Civic")
</ParamField>

### Example Request Body

```json
{
  "name": "Fusca"
}
```

## Response

<ResponseField name="id" type="long">
  The auto-generated unique identifier for the vehicle
</ResponseField>

<ResponseField name="name" type="string">
  The name of the vehicle
</ResponseField>

### Example Response

**Status Code:** `201 Created`

```json
{
  "id": 1,
  "name": "Fusca"
}
```

## Status Codes

<ResponseField name="201" type="Created">
  Vehicle successfully created
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  An error occurred while creating the vehicle
</ResponseField>

## cURL Example

```bash
curl -X POST http://localhost:8080/api/v1/vehicles \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Fusca"
  }'
```

## Controller Implementation

The create vehicle endpoint is implemented in the `VehicleController` class:

```java
@PostMapping("/vehicles")
@ApiOperation(value = "Cadastra um veículo")
@ResponseStatus(HttpStatus.CREATED)
@ResponseBody
public ResponseEntity<Vehicle> createVehicle(@RequestBody Vehicle vehicle) {
    try {
        Vehicle _vehicle = vehicleRepository.save(new Vehicle(vehicle.getName()));
        return new ResponseEntity<>(_vehicle, HttpStatus.CREATED);
    } catch (Exception e) {
        return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

**Source:** `VehicleController.java:69-80`

## Notes

- The vehicle ID is automatically generated by the database
- The name field is stored in the `vehicle` table
- Vehicles can later be associated with parts through the relationship endpoints
