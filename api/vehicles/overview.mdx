## Introduction

The Vehicles API provides endpoints for managing vehicle records in the Parts Catalog system. Vehicles can be created, retrieved, updated, and deleted, and can be associated with parts through the many-to-many relationship.

Vehicles are returned in alphabetical order by name, and you can filter the list using a search parameter.

## Base URL

```
http://localhost:8080/api/v1
```

## Vehicle Model

A vehicle object contains the following fields:

<ResponseField name="id" type="long">
  Unique identifier for the vehicle (auto-generated)
</ResponseField>

<ResponseField name="name" type="string" required>
  Name of the vehicle (e.g., "Fusca", "Gol", "Civic")
</ResponseField>

## Authentication

Currently, the API does not require authentication. All endpoints are publicly accessible.

## CORS Configuration

The API accepts cross-origin requests from `http://localhost:8081`.
