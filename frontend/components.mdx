The Parts Catalog frontend uses a component-based architecture with Vue.js 2.6.11. Each component handles a specific part of the user interface and business logic.

## Component Architecture

The application follows a typical Vue.js structure:

```
src/
├── App.vue                    # Root component with navigation
├── main.js                    # Application entry point
├── router.js                  # Route definitions
├── http-common.js             # Axios configuration
├── components/
│   ├── PartsList.vue         # List and search parts
│   ├── AddPart.vue           # Create new parts
│   ├── Part.vue              # View/edit part details
│   ├── VehiclesList.vue      # List vehicles
│   └── AddVehicle.vue        # Create new vehicles
└── services/
    ├── PartDataService.js    # Part API calls
    └── VehicleDataService.js # Vehicle API calls
```

## Component Details

<AccordionGroup>
  <Accordion title="PartsList.vue - List and Search Parts">
    **Purpose:** Display all parts with search and delete functionality.

    **Key Features:**
    - List parts in alphabetical order
    - Search by part name
    - Search by application vehicle
    - View part details
    - Delete individual parts
    - Delete all parts

    **Data Structure:**
    ```javascript
    data() {
      return {
        parts: [],              // Array of parts
        currentPart: null,      // Selected part
        currentIndex: -1,       // Selected index
        name: "",               // Search by name
        applicationVehicle: "" // Search by vehicle
      };
    }
    ```

    **Main Methods:**
    - `retrieveParts()` - Fetch all parts from API
    - `searchByName()` - Search parts by name
    - `searchByApplicationVehicle()` - Search by vehicle
    - `setActivePart(part, index)` - Select a part to view details
    - `deletePart(part, index)` - Delete a single part
    - `removeAllParts()` - Delete all parts

    **Location:** `src/components/PartsList.vue`
  </Accordion>

  <Accordion title="AddPart.vue - Create New Parts">
    **Purpose:** Form to add new parts to the catalog.

    **Key Features:**
    - Input validation
    - Form validation (net weight cannot exceed gross weight)
    - Success/error messages
    - Reset form after submission

    **Data Structure:**
    ```javascript
    data() {
      return {
        part: {
          id: null,
          name: "",
          applicationVehicle: "",
          netWeight: "",
          grossWeight: ""
        },
        submitted: false,
        errors: []
      };
    }
    ```

    **Validation Rules:**
    - Name is required
    - Net weight is required
    - Gross weight is required
    - Net weight must not exceed gross weight
    - Maximum 200 characters for text fields
    - Decimal numbers with 2 decimal places

    **Main Methods:**
    - `savePart()` - Validate and create new part
    - `newPart()` - Reset form for another entry

    **Location:** `src/components/AddPart.vue`
  </Accordion>

  <Accordion title="Part.vue - View and Edit Part Details">
    **Purpose:** Display and edit detailed information about a specific part.

    **Key Features:**
    - View part details
    - Update part information
    - Delete part
    - Link/unlink vehicles to the part
    - Display related vehicles

    **Data Structure:**
    ```javascript
    data() {
      return {
        currentPart: null,       // Part being edited
        message: "",             // Status message
        errors: [],              // Validation errors
        relatedVehicles: [],     // Vehicles linked to this part
        availableVehicles: [],   // All available vehicles
        selectedVehicleId: ""    // Vehicle to link
      };
    }
    ```

    **Main Methods:**
    - `getPart(id)` - Fetch part by ID
    - `updatePart()` - Update part with validation
    - `deletePart()` - Delete the current part
    - `linkVehicle()` - Associate a vehicle with the part
    - `unlinkVehicle(vehicleId)` - Remove vehicle association
    - `retrieveRelatedVehicles()` - Get vehicles linked to part

    **Route Parameter:** Receives `id` from route (`/parts/:id`)

    **Location:** `src/components/Part.vue`
  </Accordion>

## Component Communication

Components interact with the backend through data services:

### PartDataService

```javascript
class PartDataService {
  getAll()                    // Get all parts
  get(id)                     // Get part by ID
  create(data)                // Create new part
  update(id, data)            // Update part
  delete(id)                  // Delete part
  deleteAll()                 // Delete all parts
  findByName(name)            // Search by name
  findByVehicle(vehicle)      // Search by vehicle
}
```

### VehicleDataService

```javascript
class VehicleDataService {
  getAll()                        // Get all vehicles
  get(id)                         // Get vehicle by ID
  create(data)                    // Create new vehicle
  update(id, data)                // Update vehicle
  delete(id)                      // Delete vehicle
  deleteAll()                     // Delete all vehicles
  findByName(name)                // Search by name
  findByPart(partId)              // Get vehicles for a part
  addToPart(partId, vehicleId)    // Link vehicle to part
  removeFromPart(partId, vehicleId) // Unlink vehicle from part
}
```
