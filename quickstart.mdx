Follow these steps to set up and run the Parts Catalog application on your local machine.

## Prerequisites

Before you begin, ensure you have the following installed:

<CardGroup cols={2}>
  <Card title="Java 11" icon="java">
    Required for running the Spring Boot backend
  </Card>
  <Card title="PostgreSQL" icon="database">
    Database server with user/password: postgres
  </Card>
  <Card title="Node.js & npm" icon="node-js">
    Required for running the Vue.js frontend
  </Card>
  <Card title="Git" icon="git">
    For cloning the repository
  </Card>
</CardGroup>

<Note>
**Recommended:** Use [pgAdmin](https://www.pgadmin.org/) for easier PostgreSQL management and [Spring Tools Suite 4](https://spring.io/tools) for backend development.
</Note>

## Setup Instructions

<Steps>
  <Step title="Create PostgreSQL Database">
    Create a database named `partsCatalog` using pgAdmin:

    1. Access pgAdmin at http://127.0.0.1/pgadmin4/browser/
    2. Navigate to **Servers** → **Local** and expand
    3. Right-click on **Databases** → **Create** → **Database**
    4. Enter database name: `partsCatalog`
    5. Select owner: `postgres`
    6. Click **Save**

    <Info>
    If your PostgreSQL credentials differ from the defaults, update the connection settings in `backend/src/main/resources/application.properties`.
    </Info>
  </Step>

  <Step title="Clone the Repository">
    Clone the Parts Catalog repository to your local machine:

    ```bash
    mkdir parts-catalog-workspace
    cd parts-catalog-workspace
    git clone https://github.com/tarcnux/parts-catalog.git
    cd parts-catalog
    ```

    The repository contains two main directories:
    - `backend/` - Spring Boot application
    - `frontend/` - Vue.js application
  </Step>

  <Step title="Start the Backend">
    Run the Spring Boot backend application.

    **Option 1: Using Spring Tools Suite (Recommended)**
    1. Open Spring Tools Suite
    2. **File** → **Import** → **Maven** → **Existing Maven Projects**
    3. Select the `backend` directory as the root
    4. Right-click on `src/main/java` → **Run As** → **Spring Boot App**

    **Option 2: Using Maven Command Line**
    ```bash
    cd backend
    ./mvnw spring-boot:run
    ```

    The backend will start at **http://localhost:8080**

    <Info>
    Access the Swagger API documentation at http://localhost:8080/swagger-ui.html
    </Info>
  </Step>

  <Step title="Start the Frontend">
    With the backend running, open a new terminal and start the Vue.js frontend:

    ```bash
    cd frontend
    npm install
    npm run serve
    ```

    The frontend will start at **http://localhost:8081**

    <Warning>
    The frontend requires the backend to be running on http://localhost:8080. Make sure the backend is started before accessing the frontend.
    </Warning>
  </Step>

  <Step title="Access the Application">
    You can now access the Parts Catalog application:

    <CardGroup cols={2}>
      <Card title="Frontend Application" icon="browser" href="http://localhost:8081">
        Main application interface at http://localhost:8081
      </Card>
      <Card title="Swagger API Docs" icon="book" href="http://localhost:8080/swagger-ui.html">
        API documentation at http://localhost:8080/swagger-ui.html
      </Card>
    </CardGroup>
  </Step>
</Steps>

## Verify Installation

Test that everything is working correctly:

1. **Frontend Check**: Open http://localhost:8081 - you should see the Parts Catalog interface
2. **Backend Check**: Open http://localhost:8080/swagger-ui.html - you should see the API documentation
3. **Database Check**: Create a test part through the frontend interface

## Troubleshooting

<AccordionGroup>
  <Accordion title="npm run serve fails">
    If the frontend fails to start, try these steps:
    
    ```bash
    # Remove existing dependencies
    rm -rf node_modules package-lock.json
    
    # Clean npm cache
    npm cache verify
    
    # Update npm globally
    npm i -g npm
    
    # Reinstall dependencies
    npm install
    ```
  </Accordion>

  <Accordion title="Backend connection error">
    Ensure PostgreSQL is running and the database `partsCatalog` exists. Check the connection settings in `backend/src/main/resources/application.properties`:
    
    ```properties
    spring.datasource.url=jdbc:postgresql://localhost:5432/partsCatalog
    spring.datasource.username=postgres
    spring.datasource.password=postgres
    ```
  </Accordion>

  <Accordion title="Port already in use">
    If port 8080 or 8081 is already in use:
    
    - **Backend**: Change the port in `application.properties` by adding `server.port=8090`
    - **Frontend**: Update the port in `vue.config.js` and the API base URL in `src/http-common.js`
  </Accordion>
</AccordionGroup>
